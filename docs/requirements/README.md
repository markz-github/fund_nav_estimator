# 需求文档索引

本目录用于按功能模块维护项目需求，是需求基线和需求变更记录的入口。

## 维护规则

- 每次识别到新的需求、需求变化或新的需求细节时，必须同步更新对应功能文档。
- 如果需求影响多个模块，应分别更新相关模块文档，并在本文档的索引说明中保持入口清晰。
- 需求文档描述“要实现什么、为什么、验收标准是什么”，实现细节只保留必要约束，详细设计仍放在开发文档、数据库文档或代码中。
- 文档中的状态使用：`待处理`、`进行中`、`已完成`、`暂缓`。
- 新增功能模块时，应新增独立需求文档，并在本索引中登记。
