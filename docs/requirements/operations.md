# 运行状态与异常观测需求

## 目标

为自动任务、数据源调用和业务处理提供可观测性，让用户能查看任务执行情况和未处理异常。

## 当前状态

- 状态：进行中。
- 已有任务日志和数据异常表。
- 前端已有基金运行状态页，可查看任务日志。

## 需求清单

- 任务日志：
  - 定时任务和手动任务应记录任务名称、类型、状态、开始时间、结束时间、耗时和摘要。
  - 任务日志可记录目标类型、目标 ID 和外部任务 ID。
  - 状态统一使用 `running`、`success`、`partial`、`failed`、`skipped`。
  - `running` 只表示本地后端任务正在执行且尚未写入结束时间；`success` 表示本次后端任务完成且无失败；`partial` 表示成功与失败或跳过并存；`failed` 表示全部失败或任务级异常；`skipped` 表示没有可处理对象或业务跳过。
- 服务运行日志：
  - 控制台和文件日志应使用统一格式，包含时间、级别、进程号、线程号、logger 名称和消息。
  - Uvicorn 默认日志和项目业务日志应保持同一格式，避免同一输出中出现两套格式。
- 数据异常：
  - 数据源调用失败、业务数据缺失、外部接口失败应写入异常记录。
  - 异常记录应包含来源、数据类型、目标代码、错误信息、发生时间和解决状态。
- 前端展示：
  - 基金模块运行状态功能访问路径使用 `/fund-nav/operations`。
  - 运行状态页展示当前模块最近任务日志。
  - 任务日志支持按任务类型和状态筛选；任务类型和状态筛选项应从后端统一枚举接口获取，页面展示中文标签，筛选参数继续使用数据库保存的英文编码。
  - 任务日志支持分页查看，后端返回总条数、当前页和每页条数。
  - 运行状态页不单独展示数据异常列表；数据异常信息优先通过任务日志摘要或错误信息查看。
  - 任务日志表格应展示对应数据库 ID，便于排查和定位。
  - 其他业务页面可链接到运行状态页查看失败原因。

## 验收标准

- 用户可以在前端查看任务执行历史和失败摘要。
- 基金运行状态页只展示基金任务，不混入其他系统任务。
- 旧路径 `/operations` 应重定向到 `/fund-nav/operations`，避免历史入口失效。
- 定时任务失败不会静默失败，至少写入任务日志。
- 数据源或外部服务失败时，能定位到来源和目标对象。

## 待确认或待处理

- 异常解决、忽略和批量标记能力。
- 运行状态页分页大小配置。
- 运行中任务状态的实时刷新。
